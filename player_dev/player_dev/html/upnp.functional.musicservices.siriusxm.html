siriusxm_play_with_retry(zp)[source]¶

Attempt to play SiriusXM content. This method will re-try the play when the player can’t get the encryption key from the server. The failure manifests in a channel source transport error “Transport error ERROR_UNSUPPORTED_DRM” preceeded by “could not get key (res=1000), trying cp” On timeout, the retry verifier confirms whether the timeout was due to a failure to get the encryption key. The series of logging messages from player when this symptom is encountered: [1970-01-01 00:11:10.802] <segaudio,4> failed to open key stream code=-1 [1970-01-01 00:11:10.802] <segaudio,1> could not get key (res=1000), trying cp … [1970-01-01 00:11:30.809] <sonoscp,0> SiriusXM: http://sonos.mountain.siriusxm.com/server.php#getContentKey failed, ret = 1002 [1970-01-01 00:11:30.809] <segaudio,1> getContentKey result 1002 [1970-01-01 00:11:30.809] <chsrc,2> unrecoverable error flagged [1970-01-01 00:11:30.810] <chsrc,0> Transport error ERROR_LOST_CONNECTION [1970-01-01 00:11:30.810] <hlsaudio,1> encrypted, but no data from key URI [1970-01-01 00:11:30.810] <chsrc,2> unrecoverable error flagged [1970-01-01 00:11:30.812] <chsrc,0> Transport error ERROR_UNSUPPORTED_DRM :param sonos.client.internal.SonosZoneComponent zp:

SiriusXM bit-rate adaptation test. SiriusXM supports 35, 70, 105 and 281Kbit HLS streams Based on network bandwidth, confirm player utilizes high and low playback bit rates. Test is limited to platforms following fenway release since the algorithm to switch to higher rates does not work on legacy devices (SH4, PPC, Fenway).
