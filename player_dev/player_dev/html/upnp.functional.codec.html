test_aac_playback(**kwargs)[source]¶

verifies playback for local AAC files.

Uses ffmpeg libfdk-aac and fdkaac encoders.

ffmpeg with libfdk-aac only support CBR at specific sampling rate + bit rate combinations. It also supports very few parameters for VBR, so I used a command line front-end of libfdk-aac to handle VBR. I used the same method for heaac and heaacv2.

Parameters:zp (SonosZoneComponent) – Sonos Zone Component object
Returns:None

test_lossless_playback(**kwargs)[source]¶

Tests playback of lossless audio formats. This is a generalized method that passes in parameters of various lossless codecs, specified in LOSSLESS_TEST_PARAM, that will be used in command line calls for encoding audio.

Parameters:
  • zp (SonosZoneComponent) – Sonos Zone Component object
  • file_type ('str') – audio file format being tested
  • codec ('str') – codec used in command line specification
  • ext ('str') – file extension for file format
  • mediainfo_format ('str') – format that mediainfo returns (sometimes syntax is different from file_type)
  • bitrate_mode ('str') – constant or variable bit rate mode, used for file verification

:return:None

test_mp3_playback(**kwargs)[source]¶

Test mp3 playback for all supported sample rates in mono and stereo. Constant bitrate and variable bitrate. Uses LAME encoder. For variable bitrate, default quality is 4. LAME converts to joint-stereo (mid-side).

test_wma_playback(**kwargs)[source]¶

ffmpeg only encodes a WMA version that we do not support. Instead, downloaded files from camb-sqa-nas2IndexingCodecTestPlayback created in 2016 for a similar test, encoded using Windows WMA encoder 64. Pre-verified in mediainfo. I selected about half of them and created another directory within camb-sqa-nas2Automation.
