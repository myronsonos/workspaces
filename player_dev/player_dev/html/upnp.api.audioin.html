The AudioIn service has six state variables: AudioInputName, Icon, LeftLineInLevel, LineInConnected, Playing, and RightLineInLevel. LineInConnected can be spoofed via player debug hooks described at https://confluence.sonos.com/display/PDSW/Platform+Automation+and+Debug+Interface.

Notes:
  • Playbar does not have AudioIn service. It uses the HTAudio service.
  • Play3 and SUB have no AudioIn service.

Test Cases: Get AudioInAttributes, AudioInputName, Icon. Set them to new values. Read state variables and compare. Get LineIn levels, LeftLineInLevel and RightLineInLevel. Set them to new values. Read state variables and compare. Spoof LineInConnected. Read and compare LineInConnected state variable. Change value and verify. Un-spoof.

Playing - Will not automate. A read always times out and Device Spy does not show Playing change. Manually performing the steps described below did not cause the Playing variable to change in device spy.

Ted walked through player code: “Looks like it does get evented (but not with initial subscription, oddly). It also only updates when the iPod itself starts playing. So if you try to do anything via the Sonos CR, it doesn’t update. If you press the “play” button on iTunes/etc on the iPod, it does trigger. If the iPod is in ‘accessory connected’ mode, it doesn’t event but there’s also no actual “play/stop” button on the iPod then.

upnp.api.audioin.test_set_get_audio_in_start_stop_transmission module¶

upnp.api.audioin.test_set_get_audio_in_start_stop_transmission.AUDIO_IN_CONFIG_FILE = 'audioin.json'¶
Description:

The Start/Stop TransmissionToGroup functions gate whether or not the the AudioIn input is sent to a group coordinator for play on that group. An AudioIn device is a device with line-in or a Dock. The AVTransportURI of the coordinator must already be that of the source and the group state must be Playing in order for the StartTransmissionToGroup to play on the group. AVTransport.use_dock(dock) and AVTransport.use_line_in(line_in_device) establish the base configuration. An iPod must be in the dock for the device to be played.

There is no programmatic way to tell if transmission starts/stops. It must be heard/not-heard. Return codes and/or exceptions are all available to check for.

Test Cases to implement:

UDN of self as stand alone group UDN of self as group member
