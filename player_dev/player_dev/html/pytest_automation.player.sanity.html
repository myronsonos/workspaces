pytest_automation.player.sanity.test_standalone_parallel.test_standalone_concurrently(sanity_test_configured_standalone_players, source, toslink_source, line_in_source, track_uri, request)[source]¶

This test verifies that the signal to noise ratio for a given playback source is at least some given value. The goal of this test is to confirm that the most basic playback scenarios outputs some correct audio in a standalone configuration (i.e. no silence, no huge amounts of noise).

Parameters:
  • sanity_test_configured_standalone_players (list) – list of players that have been configured to run this test, along with the duration of the test and the max percentage of FFT windows that can have a low SNR before we fail the test
  • source (str) – the source that will be used for playback for this test
  • toslink_source – a toslink source
  • line_in_source – a line-in source
  • track_uri (str) – a URI to a music file for playback
  • request – pytest request

pytest_automation.player.sanity.test_standalone_parallel.verify_audiotap(audiotap_file, test_signal_to_noise_db, test_frequency_hz)[source]¶

Verify that the given audiotap file: 1. exists 2. has a high signal to noise ratio for the test signal frequency

Note: we do not throw asserts in this test because we want to ensure that we execute for each file for each player. Otherwise, we would stop on the first failure.

Parameters:
  • audiotap_file (str) – path to the audiotap file
  • test_signal_to_noise_db (int) – the minimum dB difference that can exist between the signal energy and noise energy before we fail
  • test_frequency_hz (int) – the frequency (in Hz) of the signal that we’re looking for
Returns:

string denoting whether or not test passed for given audiotap.
