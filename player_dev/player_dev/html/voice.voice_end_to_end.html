validate_avs_transactions(zp, file_name, log_file_name=None, ignore_spatial_data_event=True, ignore_stop_capture_directive=True, ignore_user_inactivity_report=True)[source]¶

Validate avs transactions. The verifications is based on avs directives and events specified in json spec. Currently, we can optionally ignore StopCapture directive and SpatialPerceptualData event.

Verification fails if: 1. JSON spec has avs transaction, but is not seen during test 2. JSON spec has avs transaction, it is seen during test, but it is ignored 3. JSON spec does not have avs transaction and it is not ignored.

Verification Passes if: 1. JSON spec has avs transaction and it is seen during test. 2. JSON spec does not have avs transaction and it is ignored.

Exception to above: When ‘ignore_order’ is present in avs transaction sequence, the CURRENT and NEXT (and ONLY NEXT ) event can have reversed order. For example, expectSpeech is followed by speechStarted, if ignore_order is specified in expect speech, it can occur after speechStarted and validation would not fail. Using ‘ignore_order’ 2 consecutive transactions is not supported.

:param sonos.client.internal.SonosZoneComponent zp: test zp :param str file_name: Name of voicetap file for which to perform avs

validation
Parameters:
  • log_file_name (str) – log file name to use when saving failure logs.
  • ignore_spatial_data_event (bool) – Ignore spatial data event?
  • ignore_stop_capture_directive (bool) – Ignore stop capture directive?

:return Did validation pass? :rtype: bool
