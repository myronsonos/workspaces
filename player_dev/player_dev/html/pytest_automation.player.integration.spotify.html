pytest_automation.player.integration.spotify.test_transport_control.test_verify_zp_respects_esdk_transport_limitations(sca_playback, ws_client)[source]¶

Any kind of transport control limitations for a Spotify Free experience is imposed on the player via their eSDK. All the transport policies are tracked together so they’re no value in verifying each individual action. For this test case, we know that the eSDK does not allow skipping back if you’re playing the first track in a list(free or paid), as there is no previous track. The test verifies via Muse that the ZP advertises that the “canSkipBack” action is not allowed and returns the expected Error event when we try to skip back. :param sca_playback: :param ws_client: :return:
