Given an audio file without an audio extension (.mp3, .wav, etc) set up the AVTransport and confirm the player plays the audio content. The player is expected to get a hint about the audio file encoding from the HTTP server’s response the GET request. The Content-Type header field will provide the MIME for the audio content. The field may be in mixed upper and lower case letters. The player converts the MIME content type to lower case. 1. Convert the MIME content type to mixed upper/lower case 2. Install a web file handler to serve the converted Content-Type 3. Set up a UDP watch to confirm the player selected the correct MIME type for the audio content 4. Verify the AVTransport set URI and play succeed 5. Confirm the player reports the expected MIME content 6. Confirm the player can play the audio content uninterrupted

Tests the capture and replay of SPDIF audiotaps on all HT capable devices in the testbed.

SPDIF signal sources are listed in SPDIF_TESTSIG_FILES

  1. Capture/download methods include:

  • CLI Capture method from a testrunner via curl/wget:

    wget http://<HT ZP>:1400/audio_tap?tap=spdif -O cli_audiotap.spdif

  • URL Capture method:

    http://<HT ZP>:1400/snapshotspdiftap <- capture of previous 90s input http://<HT ZP>:1400/downloadspdiftap <- download capture as custom zip

  • URL / BASE64 method:

    http://<HT ZP>:1400/snapshotspdiftap <- capture of previous 90s input http://<HT ZP>:1400/spdiftap <- download capture as custom zip via XML

  • Diag method:

    UPnP or web URL can be used to submit a diag. Audiotap is embedded in the XML but is unreadable through regular diagnostics viewer, so you need to view page source to find the <SPDIFTap> element

2. Playback of a captured audiotap can be achieved by loading the tap into /jffs on the ZP (must be same model, and ensure the file name is audiotap.spdif) supply a “reference SPDIF clock” input which matches the sample rate of the captured tap, then finally toggle anacapa to commence playback. The test checks stream type and sample rate match the original test signal played in for the capture
