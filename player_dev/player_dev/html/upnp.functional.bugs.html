By sumbitting a diagnostic the devices which support backtrace, clear the contents of the backtrace file as part of the submission. This is done at the end of the test so that subsequent tests will be able to write to the backtrace file in the event anacapad generates a backtrace.

termination_signal_generator()[source]¶

Generates a test vector based on the platform architecture. This generator will generate the zone player, the thread name, the signal which will be used to kill the thread and if it has backtrace support.

Note that each device should only ever be sent a signal once as subsequent signal doesn’t create a new backtrace file unless: 1. ip:1400/support/aggregate was called 2. backtrace file is zero bytes or not present when anacapad starts up Hence all we do here is randomized the signal type and the process to kill per device.
