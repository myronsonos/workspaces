class pytest_automation.plugins.bluetooth_plugin.SerialOnlyDiag(device)[source]¶

Bases: sonos.services.diag.Diag

This class is an override of the :cls:`~sonos.services.diag.Diag` class, retrieving diagnostic service info using a serial telnet connection, instead of through the standard web interface. This is best used when the device is in Bluetooth mode (the web interface is not available).

As more functions are needed, developers should add to this class.

Fixture to yield a test runner that is configured to start BT audio playback on a player.

Parameters:trusted_bluetooth_player_connected_to_bluetooth_source – the Bluetooth audio player and the test runner to direct audio to it

NOTE: we need to sleep for 10s between connection and specifying the audio output device to be the player when we want to start playback. Otherwise, we appear to hit some race condition on the Linux box where we don’t actually output anything to the player.
