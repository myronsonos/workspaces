pytest_automation.syssw.event_queue.test_event_queue module¶

This test verifies that the event queue is functioning as expected and also doesn’t break when abused in various ways.

Additionally, this test pokes at every proc file that generates events as appropriate per platform and verifies that the events generated are correct.

pytest_automation.syssw.event_queue.test_event_queue.test_queue_abuses(install_event_test_app, queue_abuse_args)[source]¶

Make sure that even while using a queue in a ridiculous manner, events still make it in and out in order and unaffected.

Parameters:
  • install_event_test_app (pytest.fixture) – module scoped fixture that installs/removes the event-test app on a device. Yields the device and utility object.
  • queue_abuse_args (ETOption.Arg or ETOption.NoArg paired with int or str as appropriate for the ETOption.Arg) – event-test arguments

Steps: 1. Add a number of button/cap touch presses to the event sequence -

the events don’t matter so we just send a few different presses
  1. Run the event-test app using an argument set from QUEUE_ABUSES
  2. Execute all actions in the event sequence
  3. Stop the event-test app
  4. Verify the event-test app log contains the expected events in the expected order
  5. Repeat for remaining QUEUE_ABUSES event-test arguments
pytest_automation.syssw.event_queue.test_event_queue.test_queue_overflow(install_event_test_app)[source]¶

Verifies that after overflowing a queue, it still behaves okay and will continue to pass events after the overflow occurence.

Parameters:install_event_test_app (pytest.fixture) – module scoped fixture that installs/removes the event-test app on a device. Yields the device and utility object.

Steps: 1. Run the event-test app with a combination of arguments to

make it easier to create an overflow
  1. Overflow the event queue by sending a large number of button press events as fast as possible
  2. Verify an event overflow occurred by checking the event-test app logs for an overrun event and dmesg for an overflow message
  3. Stop and run the event-test app again with no arguments
  4. Send a small number of hold events
  5. Stop the event-test app
  6. Verify the hold events are received as expected to make sure the event queue has recovered
