This fixture is a charging function that will charge the DUT until it is responsive to the automation framework.

This fixture is used by the ~charged_device fixture to make sure the device is available for testing.

This fixture may be overridden to support different implementations of the charging and is-alive indication. Pattern for override:

``` @pytest.fixture(scope=’function’) def charge_until_alive():

def charge_function(device):
# Steps to charge device until it is available for # communication

return charge_function

Usage:

def test_some_device(device, charge_until_alive):
charge_until_alive(device) …
Returns:A Function that takes a single device as an argument. This function

will charge the device until it is alive.

pytest_automation.performance.battery.conftest.pytest_addoption(parser)[source]¶

Add test specific command line options.

–playback_volume:
Used to set the volume of a device during playback test cases.
–collection_timeout:
Used to abort battery metric collection after a set time, instead of running until the device is dead. e.g. pytest -v {file} –collection_timeout 60 This command tells the collector to stop collecting data after 60 seconds. This is not a precise setting, as the sampling interval and sleep operations may cause it to go beyond this time.
–idle_discharge_minutes:
This specifies how many minutes to run the battery drain test when the device is Idle (not playing). This value is in minutes, with a default of 30.

Collect battery information on a portable device for an expected period of time.

Pre-conditions: 1. Device is fully charged 2. Portable device is paired to a bluetooth controller 3. Power coordinator is disabled on the device to prevent shutdown while idle on battery power 4. Device disconnected from power (outlets turned off)

Test Steps: 1. Collect battery data for a set period of time (default 30 minutes), saving the data to a json file.

pytest_automation.performance.battery.test_battery_life_wifi.test_gc_playback(play_portable_gc_group_audio, battery_metrics_collector)[source]¶

Collects battery percentage over time while the device is playing back a constant stream of audio until the battery is depleted. In this scenario the DUT is grouped with another device in the testbed and acts as the group coordinator.

Test Steps: 1. Charge DUT to 100% battery

(Note: group setup takes a few seconds and it is possible the
battery life testing will start with the device at 99% instead)
  1. Group the DUT with another testbed device and make it GC
  2. Set DUT volume and start audio playback
  3. Measure battery percentage and time until battery is depleted and write metrics to a json file
  4. Stop audio playback

pytest_automation.performance.battery.test_battery_life_wifi.test_idle(charged_device, kill_power_coordinator, turn_off_power_outlets, battery_metrics_collector, idle_discharge_minutes)[source]¶

Collects battery percentage over time while the device is idle and runs until the battery is depleted.

Pre-conditions: 1. charged_device() (which should be portable, see

charged_device() implementation) makes sure the DUT is charged

to 100% battery 2. kill_power_coordinator() prevents the device from going to sleep in battery mode while idle. 3. turn_off_power_outlets() disables AC power to the device

Test Steps: 1. Use battery_metrics_collector() to measure battery percentage and time until battery is depleted, writing the results to a json file

pytest_automation.performance.battery.test_battery_life_wifi.test_standalone_playback(play_portable_audio, turn_off_power_outlets, battery_metrics_collector)[source]¶

Collects battery percentage over time while the device is playing back a constant stream of audio until the battery is depleted, or the track stops. This test may be parametrized with playback volume, ‘playback_volume’

Pre-conditions: 1. play_portable_audio() provides a device charged to 100% and playing audio 2. turn_off_power_outlets() turns off AC power to the DUT

Test Steps: 1. Measure battery percentage and time until battery is depleted

and write metrics to a json file

Post-condition: battery_metrics_collector() will stop collecting when the battery has drained completely, or a timeout has been reach by using the “–timeout_collection N” flag, specifying N in seconds.
