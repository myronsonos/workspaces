networking.direct_routing.test_station_mode module¶

networking.direct_routing.test_station_mode.DIRECT_ROUTING_DISABLED_RSSI = 18¶

Station mode multicast join messages should be sent 1 every 5 seconds by the GMs. (vs. sonosnet mode is 1 every 10 seconds). These broadcast packets can be missed in the cb-mcs9-2 screen room, so we’re setting the pass threshhold in test_mc_group_join_msg_freq to ensure that it’s at least sending more than when it’s in sonosnet mode.

networking.direct_routing.test_station_mode.JOIN_MSG_MIN_SIZE = 100¶

Station mode multicast group max age is set to 120 seconds. The output of showmacs forwarding db should have seconds left close to 120 seconds for each peer.

networking.direct_routing.test_station_mode.MC_MAX_AGE = 120¶

Minimum threshhold for the max group age seen when a member is still really in the group. Per: https://confluence.sonos.com/display/PDSW/Multicast+audio+when+talking+to+an+AP a group member sends out 1 group join broadcast every 5 seconds in station mode. So if none of these probes are missed, the max age output will show anywhere between 120 to 115/114. test_mc_group_join_max_age test case will have a tolerance of missing 3 joins, but must see the 4th one. Therefore, the minimum “max age” seen must be at least 100.
