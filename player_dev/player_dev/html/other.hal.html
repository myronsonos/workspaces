Bases: sonos.workflow.fixture.WorkflowTestFixture

This class runs the PER (Packet Error Rate) test utility twice. Once with the DUT as the receiver and once with the DUT as the transmitter. The class contains methods to download the utility on each zone player, run the tool, parse the results, and remove all files from the zone players when finished.

Runs the PER tool on the transmitter ZP.

Parameters:
  • zp – The receiver ZP
  • dtype (str) – The device type (transmitter or receiver) running the tool. ‘-t’ for transmitter, ‘-r’ for receiver
  • ath (str) – Netif to use for tx/rx (ath0 or ath1)
  • channels (str) – 2.4 GHz channels to tx/rx on (CSV) <ex. 2412,2417>
  • rates (str) – Data rates to test (CSV) <ex.-54,0,1,15> Or <n> for n rates, <bg> for bg rates or <bgn> for all.
  • mcs (str) – MCS to use for signal frame <ex. -1 for 1 mbps (G) 0 for MCS 0 (N-capable only)>, <default=-1>
  • nframes (str) – Number of frames to tx/rx <default=200>
  • framesize (str) – Size of frames to tx/rx <default=220>
  • tx_power (str) – txpower values to test <ex. 18,17.5> If specified, must match on receiver and transmitter. ALTERNATIVELY: Enter <start_value>,<end_value>,<decrement> <ex. 18,10,-1> would test txpower 18,17,16,15,…,10
  • interval (str) – Delay in us between sending frames <default=10>. It is not recommended to set this too low.
  • nsignalframes (str) – Number of signal frames to send. Sending less signal frames will speed up the test but may make it less reliable.
Type:

zp: SonosZoneComponent

str tx_mac: The mac address of the applicable netif of the transmitting device

setUpFixture()[source]¶

This test limits the DUT to 1 chain while checking the chainmask registers. In order to do this, it is important to disable wifi for all devices that share a HH with the DUT (the entire testbed). If the entire HH is not disabled, it is possible that missed BPDUs coming to/from the DUT will result in a network loop.

verify_interrupts(zp, interface)[source]¶

Verifies that the number of interrupts generated by the radio is always less than the number of completions.

– NOTE: There are two general ways cases describing how interrupts are generated from the Atheros radio: interrupt mitigation enabled or disable. At the time of writing this note, Playbase is the only product with interrupt mitigation enabled. In this case, the radio will wait until one of two timers has expired before triggering an interrupt. Upon receiving the interrupt the CPU will collect, from the radio, the completion data for all packets transmitted while the timers were running. With interrupt mitigation disabled, the radio will interrupt the CPU with every packet.

Parameters:
  • zp (SonosZoneComponent) – The zone player under test
  • netif (str) – The wifi interface under test

combine_sta_id_registers(sta_id0, sta_id1)[source]¶

The STA_ID registers each contain part of the station mode MAC address. The values contained in these registers need to be reordered and and combined in order to compare them to the DUTs serial number. This method converts the string representing each value to one using big endian encoding. It also ensures that each value is prepended with any necessary zeros (for a 4 byte value, 0x94 would need to be 0x00000094). It then concats the 2 strings together.

Parameters:
  • sta_id0 (:obj:'str') – The value held in the STA_ID0 register
  • sta_id1 (:obj:'str') – The value held in the STA_ID1 register
Returns:

The combined register values
