test_genre()[source]¶

This test verifies the player assigns ID3v1 genre to an mp3 file when the genre is enveloped within an ID3v2 tag.

Player-6040 fixed a behavior on the player where the genre was missing or was not the correct genre on some mp3 audio files.

The shared device contains mp3 files with different ID3v1 genre. The test compares the player’s assigned genre (following indexing) with the expected genre maintained in a dictionary of mp3 files names to genre mappings self.TRACK_TO_GENRE.

The test does not expect to match the genre when the DUT is a non-playback or a satellite that’s “bonded” with a playbar.

upnp.functional.indexing.test_index_deep_directory_structure module¶

Indexing for Deep Directory P-12365: Index share with deep directory structure

A deep directory structure is a directory with ten (or more) subdirectories within it. A Sonos system can index a maximum of ten subdirectories deeper than the root when the share folder is added. Adding a directory with more than ten subdirectories will not throw an exception but rather only search through to the maximum depth, thus not reaching the final subdirectory. When the depth limit is reached, the UDPLogger will return a ‘Exceeded recursion limit’ string and the number(10) of directories it has searched.
