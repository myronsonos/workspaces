pytest_automation.player.automatic_update.common_automatic_update.watch_player_for_update(zp, scheduled_time, update_window=30)[source]¶

Wait until the scheduled update time and then check to see if we log the message that signals that the automatic update process has started.

Then watch for one of the following events to occur:
  1. Update prevented by music/ht playback message
  2. Update prevented by scheduled alarm message
  3. Scheduled update started message
  4. The while loop runs longer than the combined duration of the update window plus the default max expected time that a player should take to update (5 min)

If one of the update prevented messages is caught, or the while loop times out return False

If the update started, watch for the player to come online before proceeding, then return True

Parameters:
  • zp – Sonos Zone Component
  • scheduled_time – datetime object
  • update_window – integer
Returns:

String
