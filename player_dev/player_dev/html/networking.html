test_encrypt_keys_basic()[source]¶

This test verifies the following encryption key properties on a zp given the output of dumpregs on each dut ZP: 1. Since the dut zps are in station mode with ap configured for wpa2 psk aes by default, key slot 4 should have mac of ap with AES-CCM as key type 2. key slots 0-3 are reserved for GTK from the AP 3. key slots 5 until the end are reserved for direct routed sonos peers and they should be the same key for all peers since they are in the same HH. 4. all encryption keys are hexidecimal and 8 bytes long 5. inherent in checking for points 1-4, we verify that there is some key in slots 0-4 for station mode capable zps.

test_encrypted_data_ap(**kwargs)[source]¶

Test to validate the proper bits set in the encrypted data frames sent to an AP: - ZP sends encrypted data to AP with fc protected bit set - ZP sends encrypted data to AP with wep key id 0 in 802.11 header - initialization vector is incremented per frame for non-tries - When ZP disconnects from the AP, the PTK and GTK keys entries are reset

class networking.ZoneSelector(zones)[source]¶

Utility class that selects DUTs from a finite pool of zones based on string attributes. Once a zone has been “selected”, it is removed from the pool of available devices. String attributes can be any of the following: model number, arch, UDN, kernel version or serial number.
