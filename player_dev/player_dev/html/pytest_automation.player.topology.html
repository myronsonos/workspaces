pytest_automation.player.topology.test_quarantining module¶

Quarantining tests:
Verifies in a household, when a device has a software generation (swgen) different from other devices in the household, the local device quarantines other devices and vice versa. Between players of different swgen, we expect no replication, no outgoing subscription across swgens and no grouping. ZoneGroupState should be evented when quarantining state changes. See EPIC: https://jira.sonos.com/browse/SWPBL-113787 In the tests, normal player refers to the local device. Quarantined device refers to remote device whose swgen is not the same as local player

pytest_automation.player.topology.test_topology_stress.test_vanish_multiple_devices(vanish_multiple_devices)[source]¶

Verify when multiple devices go up and down, vanish list is maintained properly and no watchdogs are fired. This verifies fix for SWPBP-115421 Steps: 1. Stop all test devices 2. Start ‘local’ device 3. Start designated ‘ungraceful’ shutdown device and wait for it to appear

in ‘local’ device topology
  1. Stop designated ‘ungraceful’ shutdown device ungracefully (kill -9)
  2. Start designated ‘graceful’ shutdown device and wait for it to appear in ‘local’ device topology
  3. Bounce anacapa on designated ‘graceful’ shutdown device
  4. In order to look for a possible watchdog on local device, check for status page for 10 minutes, is status page is available, perform next iterations else fail the test.
Parameters:vanish_multiple_devices – Tuple of local device, a device that will

ungracefully shut down and a device that will gracefully shutdown
