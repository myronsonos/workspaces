pytest_automation.syssw.hdmi_compliance.conftest.zp_setup_cec_tests(*args, **kwargs)[source]¶

Fixture to setup and teardown CEC compliance tests for HDMI capable zp. TV Power Monitor and Topology Scan are features that could be in-compliant based on the timing so we turn both features off for the duration of the entire CEC test session on the ZP DUT.

The device selection is an_hdmi_capable_device() because currently the plan is to have a Hideout in one testbed and an El Rey in another for HDMI CEC compliance testbeds. They should NOT have both in one testbed because there is currently only one p8 adapter + arduino setup per testbed. If Hideout and El Rey are desired to run on the same testbed, the testing framework needs to be adjusted heavily to accommodate both at the same time.

Is module scoped and not session just in case anacapa gets bounced during the tests and clears out power monitor (false) status.
