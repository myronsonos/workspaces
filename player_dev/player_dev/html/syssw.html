Checks CPU freq transitions Steps: 1) Reboot 2) Verify that freq goes to 996000 Hz shortly after boot 3) Let device idle 4) Verify that freq goes to 396000 Hz when idle 5) Play audio 6) Verify that the freq goes to 996000 Hz while playing 7) Mute the player 8) Stop audio 9) Immediately verify freq is 996000 Hz 10) Verify that freq drops to 396000 Hz shortly after audio stops.

This test verifies that files/directories created in JFFS are maintained across reboots. To do so it creates files of random sizes on the test runner. Each DUT then creates a directory in JFFS and downloads these files from the test runner. The DUT is then rebooted and the existence of each file is verified.

This method is written to test SWPBL-40647. It verifies that, on Fenway products, JFFS is only erased when expected. In most cases, catting /proc/vnb should only show a small number of erases. If files have been added and removed from JFFS, it should show a much larger number of erases. This test uses a loose set of criteria to verify this is the case.
