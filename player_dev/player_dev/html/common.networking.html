run_basic_data_encrypt_test(dut, cap_id, chan, source_addr, dest_addr, ping_ip, ping_count, ping_size, sniff_time, key_id, wlan_fc_ds)[source]¶

Basic encrypted data sniff test does the following: 1. start wireless capture. initial capture filter will capture all frames from specified source addr. 2. start ping from DUT 3. verify all encrypted frames captured by sniffer to specified destination will have the specified key id and that the initialization vector increments per frame

Parameters:
  • dut (SonosZoneComponent) – sonos zone
  • cap_id (str) – sniffer capture interface/id
  • chan (int) – channel to sniff on
  • source_addr (str) – source mac address of ping
  • dest_addr (str) – destination mac address of ping
  • ping_ip (str) – ip to ping to
  • ping_count (int) – number of pings to send
  • ping_size (int) – payload of ping
  • sniff_time (int) – sniff time
  • key_id (int) – key id to verify in encrypted data packets
  • wlan_fc_ds (str) – DS status value to filter on

transition_zones_to_wireless_sonosnet_mode(*wireless_devices, **kwargs)[source]¶

Unplug wireless_devices from ethernet and make sure at least 1 device in the testbed that is not one of said devices has a wire. More than one device may be wired given a kwarg wired_devices which is inclusive of the kwarg first_wire device. All other devices will be disabled ( powered off)

Note

if no first wire is given then this method first attempts to find other devices not passed to it and make a first wire from one of those devices. If no other devices exist then it attempts to make a bridge first wire

Parameters:
  • wireless_devices (SonosZoneComponent) – one or more zone players to remove ethernet wires from
  • first_wire (SonosZoneComponent) – the sonos device that supplies first wire
  • wired_devices (SonosZoneComponent) – the sonos devices to be wired passed as a kwarg
