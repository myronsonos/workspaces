pytest_automation.networking.bluetooth.conftest.pytest_runtest_makereport(item, call)[source]¶

Adds a report attribute to tests that have run so that fixtures can see test result info.

A function-scoped fixture can use the request.node.report_<phase> object to see the result of that phase (passed/failed). The phase either “setup”, “call” (test run), or “teardown”.

As an example, a fixture can use the following code to check if the test that called the fixture passed or failed:

if request.node.report_call.passed:
# test passed
elif request.node.report_call.failed:
# test failed

https://docs.pytest.org/en/latest/example/simple.html#making-test-result-information-available-in-fixtures
