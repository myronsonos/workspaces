upnp.api.alarmclock.test_set_get_time_now module¶

upnp.api.alarmclock.test_set_get_time_now.PLAYLIST_URI_TEMPLATE = 'x-rincon-playlist:%s#S:%s'¶

Exercise the AlarmClock.SetTimeNow() and AlarmClock.GetTimeNow() function across the supported timezone indexes, 0-74, and verify that the returned results contain proper values. Verify that an attempt to access an invalid “index correctly fails and that the GetTimezoneRule() function still returns proper values.

Note: The last four digits of a time zone rule are the Daylight Savings Time Bias, a hex ffc4 - see ac.cxx RTimeZone serialize. The CurrentTimeZone returned from GetTimeNow() contain 0000 if the SetTimeZone AutoAdjustDst is set to False and ffc4 if True. Certain time zone rules, such as 25 and 26, are equivalent with dst set to False.

SetTimeNow() Set the current date/time used by this device. Arguments: time_now – string, desired date/time to set on device timezone – string, desired time zone rule

Exceptions: UPNP_ERROR_402_INVALID_ARGUMENTS

Returns: boolean – successfully invoked action?

GetTimeNow() Get the current time used by this device.

Returns: dictionary – current time

Response Keys: ‘CurrentTimeZone’ – rule for current device time zone ‘CurrentUTCTime’ – current UTC time for device ‘CurrentLocalTime’ – current localtime for device ‘CurrentTimeGeneration’ – current time generation

Exceptions: UPNP_ERROR_800_TIME_NOT_SET - no test in these api tests. Requires device factory reset.
